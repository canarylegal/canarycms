# Upload to canarylegalsoftware.co.uk/thunderbird

This folder is generated by `npm run publish-hosting` after signing. Upload **all files here** to your web server so they are available at:

| File | Public URL |
|------|------------|
| `updates.json` | `https://canarylegalsoftware.co.uk/thunderbird/updates.json` |
| `canary-thunderbird-{version}.xpi` | `https://canarylegalsoftware.co.uk/thunderbird/canary-thunderbird-{version}.xpi` |
| `.htaccess` | (optional but recommended on Apache) |

Thunderbird checks `updates.json` about once per day and installs newer signed builds automatically.

## WordPress on TrueNAS (Scale / CORE application)

If the site runs as a **TrueNAS application** (Docker chart), you still only need a **`thunderbird` folder next to WordPress** ‚Äî you do not create a WordPress page.

### Step A ‚Äî Find where WordPress files live

1. Open the **TrueNAS web UI** (the admin page for your NAS, e.g. `https://192.168.x.x` or `https://truenas.yourdomain`).
2. Go to **Apps** ‚Üí **Installed Applications** and click your **WordPress** (or web stack) app.
3. Open **Edit** or **Details** and look for **Storage** / **Volumes** / **Host Path** ‚Äî note any path like `/mnt/YourPool/ix-apps/.../html` or `.../wordpress`.

If that is unclear, use the **Shell** (top-right **Shell** or **System Settings ‚Üí Shell**) and run:

```bash
find /mnt -name 'wp-config.php' 2>/dev/null
```

Example output: `/mnt/tank/ix-apps/app_mounts/wordpress/html/wp-config.php`  
‚Üí WordPress root is the folder **containing** `wp-config.php` (here `/mnt/tank/ix-apps/app_mounts/wordpress/html`).  
You will create `/mnt/tank/ix-apps/app_mounts/wordpress/html/thunderbird/`.

### Step B ‚Äî Create the folder (Shell)

Replace the path with yours from Step A:

```bash
WP_ROOT="/mnt/tank/ix-apps/app_mounts/wordpress/html"   # ‚Üê change this
mkdir -p "$WP_ROOT/thunderbird"
ls -la "$WP_ROOT/thunderbird"
```

### Step C ‚Äî Copy files from your PC

**Option 1 ‚Äî SMB (easiest on Linux desktop)**  
If TrueNAS shares the pool ( **Shares ‚Üí Windows (SMB)** ):

1. In your file manager, connect to `smb://YOUR-NAS-IP/YourShare`.
2. Browse to the WordPress root from Step A (same folder as `wp-content`).
3. Open or create **`thunderbird`**.
4. Copy from your PC‚Äôs `canarycms/thunderbird-addin/hosting/`:
   - `updates.json`
   - `canary-thunderbird-{version}.xpi`
   - `.htaccess` (enable ‚Äúshow hidden files‚Äù if you do not see it)

**Option 2 ‚Äî TrueNAS UI**  
**Storage ‚Üí Datasets** ‚Üí browse to the WordPress `html` folder ‚Üí **Upload** into a new `thunderbird` dataset subdirectory (if the UI lets you create folders there).

**Option 3 ‚Äî SCP from your PC** (SSH enabled on TrueNAS):

```bash
scp -r /home/colin/canarycms/thunderbird-addin/hosting/* root@YOUR-NAS-IP:"/mnt/tank/ix-apps/app_mounts/wordpress/html/thunderbird/"
```

### Step D ‚Äî Check it works

In a browser:

- `https://canarylegalsoftware.co.uk/thunderbird/updates.json` ‚Üí JSON text  
- `https://canarylegalsoftware.co.uk/thunderbird/canary-thunderbird-1.5.6.xpi` ‚Üí download  

If you get a **WordPress 404 or theme page**, the folder is in the wrong place (often under `wp-content` by mistake) ‚Äî it must sit beside `wp-content`, not inside it.

TrueNAS WordPress charts often use **nginx** inside the container; `.htaccess` may be ignored. If the `.xpi` downloads but Thunderbird rejects it, tell your host to serve `.xpi` as `application/x-xpinstall` ‚Äî JSON usually works without extra config.

## WordPress (generic ‚Äî cPanel / FTP)

You do **not** need a WordPress page or plugin. Create a **real directory** on the server next to WordPress:

```
public_html/
  index.php          ‚Üê WordPress
  wp-content/
  thunderbird/       ‚Üê create this folder
    .htaccess
    updates.json
    canary-thunderbird-1.5.6.xpi
```

### Steps

1. In your hosting control panel (cPanel, Plesk, FTP, SFTP), open the site **document root** (often `public_html` or `www`).
2. Create folder **`thunderbird`**.
3. Upload everything from this repo folder into `thunderbird/`.
4. Verify in a browser:
   - `https://canarylegalsoftware.co.uk/thunderbird/updates.json` ‚Üí JSON (not a 404 or WordPress page)
   - `https://canarylegalsoftware.co.uk/thunderbird/canary-thunderbird-{version}.xpi` ‚Üí file download

### If WordPress catches the URL (404 or theme page)

Apache normally serves existing files before WordPress. If `/thunderbird/` still hits WordPress:

- Confirm the folder is in the **document root**, not under `wp-content/`.
- Upload `.htaccess` from this folder.
- In the **WordPress root** `.htaccess`, ensure the standard block keeps ‚Äúexisting files‚Äù first (`RewriteCond %{REQUEST_FILENAME} !-f`).

Alternative: use a subdomain (e.g. `downloads.canarylegalsoftware.co.uk`) pointing at a static folder ‚Äî then set `CANARY_TB_UPDATE_BASE_URL` when running `publish-hosting` and update `manifest.json` `update_url` to match before signing.

### nginx (not WordPress)

```nginx
location /thunderbird/ {
    alias /var/www/canarylegalsoftware/thunderbird/;
    types { application/x-xpinstall xpi; }
    default_type application/octet-stream;
}
location = /thunderbird/updates.json {
    alias /var/www/canarylegalsoftware/thunderbird/updates.json;
    default_type application/json;
    add_header Cache-Control "public, max-age=300";
}
```

## Each new release

1. Bump `version` in `../manifest.json`.
2. `npm run release` (or sign manually, then `npm run publish-hosting`).
3. Upload **new** `.xpi` and replace `updates.json` on the server (old `.xpi` files can stay for audit).

Users on a build that already includes `update_url` in the signed manifest will auto-update. The first release with `update_url` still requires a one-time manual install if users only have an older unsigned-of-update-url build.
